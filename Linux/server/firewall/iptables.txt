No kernel temos um módulo integrado ao kernel chamado netfilter, que fornece 
as funções de firewall e o iptables é a ferramenta usada para configurar esse módulo.

Comandos:
    iptables -L: Mostra as regras definidas para esse servidor
    iptables -I: adiciona regra no início da lista
    iptables -I 3: adiciona regra na posição 3 da lista
    iptables -A: Adiciona regra no final da lista
    iptables -D: Apaga uma regra do firewall
    iptables -F: Apaga todas as regras do firewall

Parâmetros:
    -p: especificar o protocolo 
    -s: especificar a origem
    -d: especificar o destino
    -i: especificar a interface de entrada
    -o: especificar a interface de saída
    --sport: especificar a porta de origem
    --dport: especificar a porta de destino
    -j: especificar a ação (ACCEPT: aceita a ação, DROP descarta sem retorno para o cliente, REJECT rejeira a ação com retorno para o cliente)

Exemplo de acição de regra: iptables -I INPUT -p ICMP -j DROP
Descrição da regra acima:
    iptables: comando base para a ferramenta iptables
    -I: adiciona regra no início da lista
    INPUT: Especifica que é regra de entrada
    -p: Especifica o protocolo
    ICMP: protocolo que vai ser usado na regra(ICMP é o protoclo usado para ping entre máquinas)
    -j: Especifica a ação
    DROP: Acão utilizada na regra(DROP descarta o pacote sem retorno para o cliente)

As regras são perdidas ao reiniciar o servidor, por isso é necessário um script
para a inicialização do firewall junto do início do servidor.

Como criar o script:
    entrar no diretório "/usr/local/bin"
    Criar o arquivo de configuração(firewall.sh por exemplo), 
        com o script 
    Na primeira linha do arquivo indicar que é um comando shell 
        com o seguinte texto:  #!/bin/bash
        o nome dessa linha (#!) é "shebang" e ela
        instrui o OS sobre qual interpretador(no caso shell) usar sem a necessidade
        de chamar um interpretador manualmente; seguido do caminho do arquivo
    Em seguida digitar a regra que deseja aplicar:
        por exemplo: iptables -I INPUT -p ICMP -j DROP
    Mudar a permissão do arquivo para poder ser lido na inicialização do OS:
        chmod +x firewall.sh
    Teste do script:
        sh firewall.sh: executa o script
        rodas o comando: iptables -L para verificar se a regra foi aplicada
        Testar a regra simulando um cliente
    Navegue até a pasta: /etc/systemd/system:
        Nessa pasta estão os arquivos correspondentes para iniciar 
        junto do systemd
    Crie um arquivo(firewall.service por exemplo)
    Digite as seguintes linhas para definir que o arquivo firewall.service
        é um serviço que deve ser iniciado jundo com o servidor:
        [Unit] // seção que descreve informações gerairs sobre a unidade
        Description=Firewall // Apenas um texto descritivo
        
        [Service] // seção principal onde é definido como o servço é iniciado, parado ou reiniciado
        ExecStart=/usr/local/bin/firewall.sh start
        ExecStop=/usr/local/bin/firewall.sh stop
        ExecReload=/usr/local/bin/firewall.sh restart

        [Install] // Diz ao sistemd o que fazer com esse service durante o boot do system
                  //Obs: Essa seção só é usada se o serviço estiver "enabled" no systemd
        WantedBy=multi-user.target //Indica que quando o sistema atingir o estado multi-user.target
                                   //o serviço será iniciado automaticamente  
        Salve o aqruivo habilite o servico e teste os comandos systemctl    

configuração de firewall de exemplo:
    Nessa parte vamos configurar um firewall para proteger o nosso servidor web e servidor 
    de DB tendo em vista que por padrão:
        SSH usa a porta 22
        HTTP usa a porta 80
        HTTPS usa a porta 443
        MySQL usa a porta 3306

    Partindo do pré-supposto que o script firewall.sh já está criado
    e que o serviço firewall.service já está configurado vamos fazer as configurações em
    cima desse arquivo

    Navegue até a pasta /usr/local/bin
    Edite o arquivo firewall.sh:
        Delete a linha com a regra que foi criada no exemplo passado
            mas mantenha o shebang
        Delete as regras pré existentes
            iptables -F // Limpa as regras de firewall
            iptables -t nat -F // Limpa as regras da tabela nat
            iptables -t angle -F // Limpa as regras da tabela mangle
        Ative o modulo do netfilter que evita ataques DoS
            echo 1 > /proc/sys/net/ipv4/tcp_syncookies // ser o echo imprimir o numero 0 ele desativa o modulo
        Libere as portas dos serviços necessários:
            iptables -A INPUT -p tcp --dport 22 -j ACCEPT
            iptables -A INPUT -p tcp --dport 80 -j ACCEPT
            iptables -A INPUT -p tcp --dport 443 -j ACCEPT
            iptables -A INPUT -p tcp --dport 3306 -j ACCEPT
        Bloqueie as demais portas
            iptables -A INPUT -p --syn -j DROP
